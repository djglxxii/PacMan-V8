## Goal

Replace the sprite converter stub with a reproducible conversion path from the
Pac-Man sprite ROM into Vanguard 8 Sprite Mode 2 pattern and row-color table
assets. This prepares the ROM data needed for T009 static sprite placement
without changing runtime sprite behavior yet.

## Scope

In scope:

- Decode all 64 16x16 2bpp sprites from `source_rom/pacman.5f`.
- Use `source_rom/82s126.4a` and the T004 palette slot map to assign sprite
  row colors in the Vanguard 8 16-color palette.
- Emit `assets/sprites_patterns.bin` as 64 Sprite Mode 2 16x16 pattern groups
  in the documented V9938 four-pattern layout.
- Emit `assets/sprites_colors.bin` as 32 sprite color-table entries
  (32 sprites x 16 rows) suitable for the VDP-B `0x7A00` table.
- Write a sprite manifest explaining which arcade sprite and CLUT each emitted
  entry came from, plus any rows where the single-color Sprite Mode 2 mapping
  had to choose a dominant visible color.
- Generate human-reviewable PNG evidence comparing source sprite pixels with
  the converted 1bpp pattern + row-color result.
- Keep conversion deterministic and driven only by source ROM/PROM data and
  authored source files.

Out of scope:

- Runtime SAT uploads, sprite placement, animation state, or visibility.
- VDP-A ghost-eye overlay sprites; this task may document candidates but does
  not need to consume VDP-A sprite tables yet.
- Collision detection, movement, Pac-Man direction changes, fruit scheduling,
  or audio.
- Changes to the accepted T006/T007 maze and HUD display.

## Implementation notes

The existing `vanguard8_port/tools/conv_sprites.py` is a stub invoked by
`tools/pack_rom.py`. Replace it with the same local style used by the T005
tile converter: load the sprite ROM, decode the MAME layout, load the 64 CLUT
rows, map PROM colors through `assets/src/palette_map.md`, write binary assets,
and generate evidence.

For the 16x16 Sprite Mode 2 pattern generator, each arcade sprite should occupy
four consecutive 8x8 pattern slots in the V9938 layout. Store rows in the order
top-left, top-right, bottom-left, bottom-right so a SAT pattern number aligned
to a four-pattern group selects the intended 16x16 shape.

Because Sprite Mode 2 supports one visible color per sprite row, pack the
pattern bit as `1` for any non-zero arcade plane value. Choose the row color
from the dominant non-zero plane value after CLUT/palette-slot mapping. Rows
with no visible pixels should use color 0 with the invisible bit set. Rows that
contain multiple non-zero colors must be listed in the manifest so T009 can
decide whether a second overlay sprite is needed for a given actor.

The T008 binary assets are conversion deliverables only. T009 will upload
sprite tables and place Pac-Man/ghosts in static screen positions.
